# Creation Workflow

## Inputs

Infer from context when possible:

```text
purpose:
duration:
main role or point of view:
event or theme:
mood:
vocal or instrumental:
product or scene constraint:
output format:
```

If the user gives none of these, create a minimal useful draft based on project context and state the assumption.

## Output Format

For Suno generation, output:

```text
目标判断：
Style Prompt：
Lyrics：
生成建议：
剪辑/使用建议：
下一轮复盘点：
```

For simple requests, shorter output is fine, but always include copy-ready `Style Prompt` and `Lyrics` or explain why not.

## Short-Video Hook Pattern

Use this for TikTok/Douyin, Reels, Shorts, product clips, and pet/lifestyle videos:

```text
[Hook, vocals start immediately]
line 1: concrete event
line 2: role reaction
line 3-6: real details
line 7: turn the small thing into a memorable conflict
line 8: repeatable ending line
[End]
```

Keep lines short enough for subtitles.

## Full Song Pattern

Use only when the user asks for a complete song or when a short hook has already tested well:

```text
[Verse 1]
[Pre-Chorus]
[Chorus]
[Verse 2]
[Bridge]
[Final Chorus]
[End]
```

Do not default to a full song for short-video work.

## Variation Strategy

When useful, provide A/B/C versions:

```text
A stable version: mainstream, clean, likely usable
B reinforced version: clearer hook, stronger rhythm, more direct
C exploratory version: different genre or mood
```

Keep changes intentional, not random.
